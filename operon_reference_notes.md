# Operon Reference Notes Reference URL: https://operonflooring.com.au/products Audit date: 2026-05-15 Read-only boundary: - Operon Flooring is separate from Oz Timber Floor.
- Operon was inspected only as a visual and catalogue-structure reference.
- No Operon files, folders, product data, quote flow, floorplan tool, pricing logic, backend logic or API keys were modified or copied. ## Useful For Oz The Operon products page has a clear decision path: - Start with category selection.
- Let customers filter by brand, colour or thickness.
- Keep the product-selection state visually obvious.
- Keep product browsing focused on choosing the right flooring direction before the next action.
- Explain what customers should check before selecting a product. Useful Oz adaptations: - Use category-first browsing on `/products/`.
- Use clear category thumbnails and range cards.
- Keep `/ranges/` as the full range library.
- Let customers search/filter ranges without overwhelming the first product page.
- Show selected product/range context on contact enquiries in customer-friendly wording.
- Keep CTAs clear: view details, check stock, request supply price, request supply + install quote. ## What Oz Should Improve Oz Timber Floor should be simpler and more traditional than Operon: - Oz should not lead customers into a quote app.
- Oz should focus on supplier catalogue browsing, colour comparison and enquiry.
- Oz should avoid"fastest path to quote" wording.
- Oz should show richer supplier/range data where verified from Eco,,, and source pages.
- Oz should show local, verified product images rather than generic or mismatched imagery.
- Oz should use phone/email/contact pathways prominently because Oz is a supplier and contractor. ## Must Not Be Copied Do not copy from Operon: - Quote flow.
- Start quote CTAs as the main product journey.
- Floorplan upload flow.
- Quote review flow.
- Pricing logic.
- Checkout logic.
- Backend or API behaviour.
- Operon product data.
- Operon code, components or files. ## Practical Direction For Oz Oz can borrow the high-level browsing clarity: 1. Choose category.
2. Find a range.
3. Compare colours.
4. Send an enquiry. Oz must keep the customer journey as catalogue/enquiry only, with no checkout, no instant pricing and no internal pricing exposure.
