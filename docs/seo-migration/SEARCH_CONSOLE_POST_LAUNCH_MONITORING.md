# Search Console Post-Launch Monitoring

## Day 0
- Submit `https://oztimberfloor.com.au/sitemap.xml`
- Submit `https://oztimberfloor.com.au/sitemap-keyword-targets.xml`
- Inspect homepage URL
- Inspect:
  - `/hybrid-flooring-sydney/`
  - `/laminate-flooring-sydney/`
  - `/engineered-timber-flooring-sydney/`
  - `/solid-timber-flooring-sydney/`
  - `/vinyl-flooring-sydney/`
  - `/timber-flooring-installation-sydney/`
  - `/floor-levelling-sydney/`
- Inspect a sample of old redirected URLs:
  - `/hybrid/`
  - `/laminate/`
  - `/floor-levelling/`
  - `/blogs/`
  - `/product-category/laminate/infinite-laminate/`

## Day 1–3
- Check `Page indexing`
- Check `Not found (404)`
- Check `Page with redirect`
- Check `Duplicate, Google chose different canonical`
- Check sitemap processing and submitted URL counts
- Re-test old high-priority WordPress URLs manually if any errors appear

## Day 7
- Compare clicks, impressions, CTR and average position against pre-launch baseline
- Check whether old URLs are dropping out and new URLs are replacing them correctly
- Fix any redirect or canonical mismatches immediately
- Review range and guide pages with the highest impressions first

## Day 14–30
- Monitor ranking volatility by service, category and guide cluster
- Watch for unexpected 404 spikes
- Watch for canonical drift
- Expand or refine content only after migration stability is confirmed

## Priority Alerts
Treat these as fast-response issues:
- old high-intent URLs returning `404`
- important old URLs redirecting to homepage instead of a relevant target
- Netlify or preview URLs appearing in Search Console indexing reports
- canonical conflicts on category or guide pages
